git配置

  •   新建一个 repository 流程跟上文一样 直到copy主题停止
  • 修改hexo配置文件_config.yml 注：url网址 deploy 配置git地址
  • 然后把你自己写的html放在themes/your_themes_name/source/这个目录里的文件，在你每次hexo g的时候都会全部复制，所以接下来就简单了，只需要把你的项目放到这个里面就OK了（自定义的index.html得在哈，也就是source目录做你项目的根目录）
  • 在你自己写的html文件最上面写上：layout: false — 上传
  • 防止html被 hexo的source文件夹中会默认全部渲染，在hexo根目录下的_config.yml 配置skip_render:
    ###git技巧
  • 在 GitHub.com 上删除某个 Repository 中的某个文件夹 这里指 s 文件夹

    git rm -r –cached s
    git commit -m ‘delete s dir
    git push -u origin master
